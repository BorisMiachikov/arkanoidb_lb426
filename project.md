# Техническое задание: Игра в жанре Arkanoid/Breakout (Rust + Bevy 0.15)

## 1. Общая информация

**Название проекта:** arkanoidb_lb426
**Жанр:** Аркада / Breakout
**Язык разработки:** Rust (stable)
**Движок:** Bevy 0.15
**Физика:** Кастомная (AABB-коллизии, без Rapier)
**Репозиторий:** https://github.com/BorisMiachikov/arkanoidb_lb426

---

## 2. Описание игры

Игрок управляет платформой (ракеткой) в нижней части экрана.
Цель — отбивать мяч и уничтожать все блоки на уровне.

Игра заканчивается:
- **Победой** — при уничтожении всех блоков → `LevelComplete`
- **Поражением** — если мяч падает за нижнюю границу при 0 жизнях → `GameOver`

---

## 3. Основной игровой цикл

1. Главное меню (`MainMenu`) → Enter/Space → `Playing`
2. Инициализация уровня (`OnEnter(Playing)` → `spawn_level_entities`)
3. Мяч появляется прилипшим к ракетке (`BallStuck`)
4. Запуск мяча пробелом или движением ракетки
5. Обработка ввода → движение → коллизии → эффекты
6. Проверка условий победы/поражения
7. Переход к следующему уровню или рестарт

---

## 4. Игровые сущности

### 4.1 Ракетка (Paddle)
- Управляется: A/D или ←/→, скорость 400 px/s
- Границы вычисляются динамически по `Collider.half_width` — корректно при увеличении
- Эффекты: PaddleGrow (×1.5 на 10 сек), StickyPaddle (прилипание на 10 сек), GunPaddle (пулемёт на 15 сек)

### 4.2 Мяч (Ball)
- Размер: 20×20, начальная скорость: vx=200, vy=350 (×`BallSpeedMultiplier`)
- Отскакивает от: стен, ракетки, блоков, НЛО
- Угол отскока от ракетки зависит от точки контакта
- Эффект: BallGrow (×1.5 на 10 сек)

### 4.3 Блоки (Bricks)
- Normal: 1 удар, 100 очков
- Strong: 2 удара, 200 очков (красный)
- 25% шанс дропа бонуса при уничтожении

### 4.4 Бонусы (Power-ups)

| Тип | Цвет | Эффект | Длительность |
|-----|------|--------|-------------|
| PaddleGrow | зелёный | ракетка ×1.5 | 10 сек |
| StickyPaddle | жёлтый | мяч прилипает | 10 сек |
| BallGrow | голубой | мяч ×1.5 | 10 сек |
| GunPaddle | оранжевый | пулемёт (Ctrl) | 15 сек |

Падают вниз, активируются при касании ракеткой. HUD показывает активные бонусы с таймерами.

### 4.5 НЛО (UFO)
- Движутся горизонтально, отражаются от стен и блоков
- Уничтожаются за **2 удара** мячом или снарядами
- После уничтожения **респавнятся** случайно выше блоков `y ∈ [200, 270]` или ниже `y ∈ [−120, 0]`
- Конфигурируются в `LevelConfig`: позиция, скорость, интервал бомб

### 4.6 Бомбы (Bomb)
- Сбрасываются НЛО по таймеру, падают со скоростью −220 px/s
- При попадании в **ракетку** → потеря жизни (или GameOver)
- При попадании в **блок** → бомба исчезает (блок не повреждается)
- Исчезают за нижней границей экрана (y < −350)

### 4.7 Снаряды пулемёта (Bullet)
- Спавнятся при зажатом Ctrl (левый или правый), если активен GunPaddle
- Скорость +520 px/s (вверх), размер 4×14
- При попадании в блок: 1 урон блоку, снаряд исчезает
- При попадании в НЛО: 1 урон НЛО, снаряд исчезает
- Исчезают при выходе за верхнюю границу

## 5. Уровни

Данные уровней — статический массив `LEVELS` в `src/resources/level_data.rs`:

| Уровень | Паттерн | НЛО | Скорость | Бомба (интервал) |
|---------|---------|-----|---------|-----------------|
| 1 | Классический (10×5) | 0 | ×1.0 | — |
| 2 | Шахматный | 1 | ×1.25 | 4 сек |
| 3 | Пирамида | 2 | ×1.5 | 3 сек |

Формат сетки: `&[&[u8]]`, где `0` = пусто, `1` = Normal, `2` = Strong.

---

## 6. Физика и коллизии

Подход: **AABB** (Axis-Aligned Bounding Box).

| Пара | Эффект |
|------|--------|
| Ball ↔ Wall | отскок |
| Ball ↔ Paddle | угловой отскок |
| Ball ↔ Brick | отскок + урон блоку |
| Ball ↔ UFO | отскок + урон НЛО |
| UFO ↔ Brick | разворот НЛО (блоки непроходимы) |
| Bonus ↔ Paddle | подбор бонуса |
| Bomb ↔ Paddle | потеря жизни |
| Bomb ↔ Brick | бомба исчезает |
| Bullet ↔ Brick | урон блоку, снаряд исчезает |
| Bullet ↔ UFO | урон НЛО, снаряд исчезает |

---

## 7. Управление

| Клавиша | Действие |
|---------|----------|
| A / ← | ракетка влево |
| D / → | ракетка вправо |
| Пробел | запуск мяча |
| Ctrl (Left/Right) | стрельба пулемётом (если активен GunPaddle) |
| Escape | пауза / снять паузу |
| `*` Numpad | **[DEBUG]** следующий уровень |

---

## 8. Состояния игры (Game States)

```
Startup → MainMenu → Playing → LevelComplete → Playing (следующий уровень)
                             ↘ GameOver → Playing (рестарт)
```

| Состояние | Описание |
|-----------|----------|
| `MainMenu` | Главное меню (ENTER/SPACE → старт) |
| `Playing` | Активная игра |
| `GameOver` | Игра окончена (показывает счёт) |
| `LevelComplete` | Уровень пройден (показывает счёт и уровень) |

**Пауза** — через ресурс `Paused(bool)`, Escape переключает. Экран PAUSE показывается поверх игры без изменения `GameState`.

## 9. Архитектура (Bevy ECS)

### Плагины

| Плагин | Ответственность |
|--------|----------------|
| `GameplayPlugin` | Правила игры, ресурсы, победа/поражение, пауза |
| `PhysicsPlugin` | Ввод, движение, все коллизии, бонусы, НЛО, пулемёт |
| `UiPlugin` | HUD + экраны состояний |
| `LevelPlugin` | Камера, спавн/очистка уровня |

### Компоненты (только данные)

```
Velocity { x, y }
Collider { half_width, half_height }
Paddle { speed }
Ball { radius }  +  BallStuck (маркер)
Brick { brick_type, health, score_value }
Bonus { bonus_type }
PaddleGrowEffect / StickyEffect / BallGrowEffect / GunPaddleEffect { timer, ... }
Bullet (маркер)
Ufo { speed, direction, bomb_timer, health }
Bomb { damage }
Wall (маркер)
LevelEntity (маркер для cleanup)
```

### Ресурсы

```
GameState (States enum): MainMenu | Playing | GameOver | LevelComplete
Score { value }
Lives { count }
CurrentLevel { number }
BallSpeedMultiplier(f32)
DebugSkipPending(bool)
Paused(bool)
```

### Структура `src/`

```
src/
├── main.rs
├── app.rs
├── plugins/       gameplay, physics, ui, level
├── components/    ball, brick, bonus, bonus_effects, bomb, bullet, collider,
│                  level_entity, paddle, ufo, velocity, wall
├── systems/       input, movement, collision, gameplay, bonus, gun, ufo
├── resources/     game_state, score, level_data
└── setup/         camera, level
```

## 10. UI

- **HUD** (всегда виден): счёт (слева), уровень (центр), жизни (справа)
- **Бонусы**: строка под HUD, показывает активные эффекты с оставшимся временем
- **Экраны состояний**: MainMenu, GameOver (счёт), LevelComplete (уровень + счёт), Pause
- Текст на латинице (дефолтный шрифт Bevy не поддерживает кириллицу)

## 11. Требования к коду

- Чистая ECS-архитектура (Bevy)
- Компоненты — только данные, без логики
- Системы — только логика, без состояния
- Минимальная связанность через ECS (Query, Res)
- Без сторонних физических библиотек
