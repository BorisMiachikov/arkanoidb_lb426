# Прогресс разработки: Arkanoid (Rust + Bevy 0.15)

## ✅ Этап 0: Подготовка проекта
- [x] Rust toolchain (stable)
- [x] Создан проект (`Cargo.toml`, `bevy = "0.15"`, `rand = "0.8"`)
- [x] Настроено окно 800×600, фон — тёмно-синий
- [x] Пустая сцена запускается

## ✅ Этап 1: Базовая сцена и сущности
- [x] 2D камера (`Camera2d`)
- [x] Ракетка (Paddle) — синий прямоугольник внизу экрана (y = -250)
- [x] Мяч (Ball) — белый круг (r = 10)
- [x] Стены: левая, правая, верхняя (нижней нет — зона проигрыша)

## ✅ Этап 2: Управление игроком
- [x] A/D и ←/→ — движение ракетки (400 px/s)
- [x] Границы вычисляются динамически из `Collider.half_width` (корректно при PaddleGrow)

## ✅ Этап 3: Движение мяча
- [x] Компонент `Velocity { x, y }`
- [x] `apply_velocity_system` с `dt.min(0.05)` против туннелирования
- [x] Мяч стартует прилипшим к ракетке (`BallStuck`)
- [x] Пробел или движение ракетки — запуск мяча
- [x] Направление запуска зависит от движения ракетки

## ✅ Этап 4: Коллизии (AABB)
- [x] Ball ↔ Wall — отскок с корректировкой позиции внутри стены
- [x] Ball ↔ Paddle — угловой эффект, `hit_factor.clamp(-0.8, 0.8)`, мин. vy = speed×0.4
- [x] Нормировка скорости после каждого отскока

## ✅ Этап 5: Блоки
- [x] Сетка блоков из конфига уровня (`grid: &[&[u8]]`)
- [x] Normal (1 удар, 100 очков), Strong (2 удара, 200 очков, красный)
- [x] Очки начисляются в `Score`
- [x] 25% шанс дропа одного из 4 бонусов при уничтожении блока

## ✅ Этап 6: Игровая логика
- [x] Мяч y < −320 → потеря жизни, мяч → BallStuck над ракеткой
- [x] 0 жизней → `GameOver`
- [x] Все блоки уничтожены → `LevelComplete`
- [x] Enter/Space: рестарт из GameOver (сброс счёта/жизней) и LevelComplete
- [x] `LevelEntity` маркер + `cleanup_level` при `OnExit(Playing)`

## ✅ Этап 7: Система уровней
- [x] 3 уровня: классический, шахматный, пирамида
- [x] `BallSpeedMultiplier`: x1.0 → x1.25 → x1.5
- [x] `CurrentLevel` ресурс, wraparound после последнего уровня
- [x] Данные уровней — статические `LevelConfig` в `level_data.rs`

## ✅ Этап 8: Бонусы (Power-ups)
- [x] Дроп из блоков — 25% шанс, случайный тип из 4
- [x] Падение вниз через `Velocity`
- [x] Подбор ракеткой (AABB Bonus ↔ Paddle)
- [x] PaddleGrow — ракетка ×1.5 на 10 сек
- [x] StickyPaddle — мяч прилипает при касании на 10 сек
- [x] BallGrow — мяч ×1.5 на 10 сек
- [x] GunPaddle — пулемёт на 15 сек (реализован в Этапе 14)
- [x] Откат всех эффектов по таймеру

## ✅ Этап 9: НЛО (UFO)
- [x] Горизонтальное движение, отражение от боковых стен
- [x] UFO ↔ Brick — НЛО не проходит сквозь блоки, разворачивается
- [x] Ball ↔ UFO коллизия (AABB, мяч отражается)
- [x] НЛО уничтожается за **2 удара** мячом (или снарядами)
- [x] После уничтожения — **респавн** случайно выше блоков `y ∈ [200, 270]` или ниже `y ∈ [−120, 0]`
- [x] Уровень 2: 1 НЛО (скорость 130, бомба 4 сек); уровень 3: 2 НЛО (170, 3 сек)

## ✅ Этап 10: Бомбы
- [x] Периодический сброс бомб по таймеру (настраивается в `LevelConfig`)
- [x] Падение вниз (`Velocity`, скорость −220 px/s)
- [x] Bomb ↔ Paddle — потеря жизни, BallStuck или GameOver
- [x] Bomb ↔ Brick — **бомба исчезает** при попадании в блок
- [x] Очистка бомб за нижней границей (y < −350)

## ✅ Этап 11: UI
- [x] HUD: счёт (слева), уровень (центр), жизни (справа)
- [x] Строка активных бонусов с оставшимся временем (`[PADDLE+ 8.3s]` и т.д.)

## ✅ Этап 12: Game States (экраны)
- [x] MainMenu — заставка, ENTER/SPACE → старт
- [x] GameOver — экран с итоговым счётом, ENTER → рестарт
- [x] LevelComplete — экран с уровнем и счётом, ENTER → следующий уровень
- [x] Pause — оверлей через ресурс `Paused(bool)`, ESC переключает (не меняет GameState)
